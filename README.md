# 💼 HR 差旅自動報銷系統

本專案為一個網頁版差旅報銷系統，旨在將傳統人工報銷流程數位化，透過規則判斷與流程分流，提升企業審核效率並降低錯誤率。

---

## 🎯 專案目標

優化傳統報銷流程中以下問題：
- 流程繁瑣、耗時
- 人工審核容易出錯

透過系統化設計，使報銷流程自動化與標準化。

## 🧩 系統流程設計

本系統將報銷流程拆分為「申請 → 審核 → 執行 → 報銷 → 結案」五個階段，並透過規則引擎進行自動化判斷。

---

### 1️⃣ 報銷申請（Application）
使用者提交報銷申請單：
- 報銷類別
- 預估金額
- 申請用途
- 基本資料

---

### 2️⃣ 初步審核（Manager Review）
系統檢查：
- 資料是否完整
- 是否符合基本規範

→ 通過後進入後續流程

---

### 3️⃣ 任務執行（Business Execution）
使用者進行出差或任務執行

---

### 4️⃣ 報銷提交（Expense Submission）
填寫實際報銷內容：
- 實際金額
- 憑證資料
- 地點與時間

---

### 5️⃣ 風險評估（Risk Engine ⭐核心）

系統依據規則進行風險判斷：

#### 🔴 高風險條件
- 實際金額 > 申請金額
- 超過公司規範上限
- 憑證缺失或不一致
- 資料異常（時間/地點不合理）

→ 流程：主管審核 → 會計審核

---

#### 🟢 低風險條件
- 符合申請範圍
- 憑證完整
- 符合公司規範

→ 流程：會計審核

---

### 6️⃣ 結案（Completion）
會計審核完成後，流程結束
